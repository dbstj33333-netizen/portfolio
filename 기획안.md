# UI/UX 디자이너 포트폴리오 인트로 웹사이트 — 기획안

> Figma 디자인(이미지 1~6)을 기준으로 한 인터랙티브 포트폴리오 메인 페이지 구현 기획.

## 전체 콘셉트

- 핵심 문구: **“Make it Clear.”**
- 전직 안경사/검안사 경험에서 출발 → 흐릿한 문제를 선명하게 보고 사용자의 불편을 해결하는 UI/UX 디자이너 콘셉트.
- 톤: 밝고 미니멀. 흰색 + 연한 하늘색 배경, 선명한 블루 컬러 중심.
- 과한 장식보다 **여백 · 흐림 효과 · 얇은 선 · 부드러운 스크롤 전환** 중심.
- 목표: 단순 정적 페이지가 아니라 스크롤에 따라 화면이 자연스럽게 전환되는 인터랙티브 포트폴리오.

## 기술 조건

- HTML / CSS / JavaScript 기반.
- 복잡한 스크롤 애니메이션은 GSAP + ScrollTrigger 사용 가능.
- 파일 구조:
  ```
  index.html
  css/style.css
  js/main.js
  images/
  ```
- 반응형: **데스크톱 1920px 우선**, 이후 태블릿/모바일 대응 구조.
- class 이름은 의미 있게, CSS/JS에는 섹션별 주석.

## 공통 스타일

- 전체 배경: 거의 흰색에 가까운 연한 블루/화이트 그라데이션.
- 메인 컬러: 선명한 블루 계열.
- 주요 그래픽: 얇은 라인과 큰 원형 라인.
- 폰트: 얇고 깔끔한 산세리프 (페이퍼로지/Paperlogy).
- 영문 타이틀은 크고 얇게, 한글 설명은 작고 정돈되게.
- 화면 전체는 100vh 단위 섹션처럼 보이되 스크롤 애니메이션으로 연결.

## [1번 화면] 첫 화면 / 흐릿한 인트로

- 중앙 큰 문구 (흐릿하게):
  - 윗줄: `UIUX DESIGNER PORTFOLIO`
  - 아랫줄: `Make it Clear.`
- 주변에 사용자를 표현하는 키워드들 흐릿하게 배치.
- 하단 중앙: `SCROLL` 텍스트 + 아래 방향 화살표(3단, 연한 파랑 → 진한 파랑).
- 첫 화면 전체는 안개 낀 듯 blur 처리.
- 핵심: “아직 선명하지 않은 문제” 표현. 텍스트는 흐릿하되 이후 마우스 효과로 일부가 선명해질 수 있어야 함.

## [2번 화면] 마우스 오버 시 렌즈 효과

- 마우스를 따라다니는 큰 원형 렌즈.
- 원 안쪽만 텍스트가 선명, 원 밖은 계속 흐릿.
- 원에는 아주 얇은 블루 라인 border. 부드럽게 마우스를 따라 이동.
- 렌즈 안: `UIUX DESIGNER PORTFOLIO`, `Make it Clear.` 가 선명한 블루.
- 구현: 동일 텍스트 2겹(흐림 배경 레이어 / 렌즈 안 선명 레이어), 선명 레이어는 clip-path 또는 mask로 원형 영역만 노출. 마우스 위치를 CSS 변수/JS로 업데이트.

## 스크롤 애니메이션 전체 흐름

1. 흐릿한 인트로.
2. 마우스 이동 시 원형 렌즈 안쪽만 선명.
3. 스크롤 시작 → 렌즈 원이 확대되며 INDEX로 전환.
4. INDEX에서 큰 원형 라인이 배경 요소로 유지.
5. 스크롤 → 원형 라인이 오른쪽으로 이동하며 About Me로 전환.
6. 스크롤 → 원형 라인이 깔때기로 변형되며 Why UI/UX로 전환.
7. 키워드들이 깔때기 아래로 쏟아짐.
8. 스크롤 → 깔때기가 좌우로 펼쳐지며 Project 시작 화면으로 전환.

> 중요: 화면이 뚝뚝 끊기지 않고, **하나의 큰 그래픽 요소가 계속 변형되며 이어지는 느낌**.
> 원형 렌즈 → 큰 원형 라인 → 오른쪽 원형 배경 → 깔때기 → 펼쳐지는 라인 순으로 자연스럽게 연결.
> 모든 애니메이션은 과하지 않고 부드럽게.

4. GSAP ScrollTrigger 연결
5. 1→3번 원 확대 전환
6. INDEX 화면
7. About Me 화면
8. Why UI/UX 깔때기 화면

10. 반응형 기본 대응

## 주의사항

- Figma 디자인과 다른 임의 디자인으로 바꾸지 말 것.
- 여백, 큰 타이포그래피, 얇은 선, 연한 블루 컬러감 유지.
- 텍스트가 너무 작거나 붙어 보이지 않게.
- position / z-index / overflow 신중히 → 애니메이션으로 레이아웃 깨지지 않게.
- 스크롤 중 가로 스크롤 금지.
- 데스크톱 완성도 우선, 반응형은 이후.
- 복잡한 부분은 구조 먼저 → 세부 애니메이션 조정.
