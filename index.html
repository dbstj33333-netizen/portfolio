Background 모카클래스 웹 UI/UX 개선

복잡하게 나열된 클래스 탐색 구조

수업 형태, 지역, 카테고리, 이용 목적 등 서로 다른 탐색 기준이 같은 위계로 반복 노출되어 사용자가 원하는 클래스를 어떤 경로로 찾아야 하는지 파악하기 어려웠습니다.

기존 사이트의 문제점
  • 혼재된 탐색 기준
  • 반복되는 클래스 목록
  • 불명확한 콘텐츠 선정 기준
  • 과도하게 강조된 필터 영역
Problem & Goal 모카클래스 웹 UI/UX 개선

핵심 문제 3가지

01

탐색 기준 혼재

서로 다른 기준의 카테고리가 같은 위계로 노출됨

02

반복되는 콘텐츠

유사한 클래스 목록의 차이와 목적이 명확하지 않음

03

필터의 높은 시각적 비중

필터가 실제 클래스 콘텐츠보다 먼저 강조됨

Goal 복잡한 정보 구조를 정리해 사용자가 원하는 클래스를 쉽고 빠르게 찾을 수 있는 탐색 경험을 설계했습니다.

⋮

문제 해결 방안 시스템치과 웹 UI/UX 개선

AS-IS

전후 사진, 할인 혜택, 캐릭터 이미지 중심의 구성으로 병원의 전문성과 신뢰감이 약하게 전달되고 있었습니다.

TO-BE

의료진, 진료 시스템, 병원 공간, 치료 과정 중심으로 재구성해 사용자가 병원의 전문성을 빠르게 이해할 수 있도록 개선했습니다.

  • 2

    진료 및 시스템 강점 소개

  • 3

    환자 안심 콘텐츠 구성

    치료 과정과 진료 시스템을 단계적으로 보여주어 사용자 불안감 완화

  • 4

    예약 전환 유도

    고정 플로팅 메뉴와 예약 CTA를 배치해 상담·예약 접근성 강화
